# 定时任务与内容更新规则

## 原则

- 网站每天设置一个固定更新窗口，不追求小时级实时性。
- AI 只处理灵感收件箱并生成草稿，不能自动设置“已发布”或公开级别。
- 公开内容必须经过人工核验；任一发布门禁失败时保留上一个可用版本。
- 定时同步与妙搭发布分离。同步可以自动运行，妙搭发布在第二阶段完成自动化与回滚验证前保留人工执行。

## 当前已启用

### 每日内容同步

- 时间：每天 18:00（Asia/Shanghai）。
- GitHub cron：`0 10 * * *`，GitHub Actions 使用 UTC。
- 入口：`.github/workflows/sync-content.yml`。
- 处理范围：
  - 把灵感收件箱中“待处理”的记录转换为待审核草稿。
  - 读取已经人工审核并标记为“已发布”的内容。
  - 执行价值、时效、事实和下次复核时间门禁。
  - 生成公开数据集和内容图片。
  - 通过测试、类型检查、构建和公开信息扫描后提交到默认分支。
- 失败策略：不提交半成品，不覆盖 last-good 数据集，不自动降低审核标准。

### 人工快速通道

以下情况不等待当天 18:00 的任务，审核完成后手动运行 **Actions > Sync public content > Run workflow**：

- 新内容需要当周发布。
- 已发布内容需要紧急纠错或下架。
- 发布门禁修复后需要重新验证。
- 灵感收件箱积压需要提前生成草稿。

## 每日运营节奏

| 时间 | 负责人 | 动作 | 结果 |
| --- | --- | --- | --- |
| 白天 | 团队成员 | 向灵感收件箱添加链接、项目、Skill、案例或需求 | 待处理记录 |
| 每天 18:00 | GitHub Actions | 处理收件箱、校验已发布内容、生成公开数据集 | 草稿和同步结果 |
| 18:00 后 | 内容负责人 | 审核事实、价值、网络要求、图片、版权和复制块 | 已发布或退回修改 |
| 次日 18:00 前 | 内容负责人 | 完成审核；紧急内容可手动同步并发布妙搭 | 当日或次日更新上线 |

## 复核周期

- 高波动 AI 信号、模型版本和快速迭代项目：7 天。
- 工具、Skill、CLI 安装和配置方法：30 天。
- 稳定方法、团队案例和工作原则：90 天。
- 超过“下次复核时间”的已发布内容会阻断同步，必须更新、重新核验或下架。

## 妙搭发布

审核和同步完成后，在可信的公司电脑上执行：

```powershell
cd apps/web
$env:MIAODA_APP_ID = "app_17a5paexd1j"
$env:MIAODA_PUBLIC_ORIGIN = "https://g2b5dhmahw.aiforce.cloud"
npm run build:miaoda
lark-cli apps +html-publish --app-id app_17a5paexd1j --path ./dist --as user
```

发布后必须检查：首页轮播自动切换、快速匹配可点击、反馈弹窗可打开、复制按钮可写入剪贴板、桌面与移动端控制台无错误。
