# 第二阶段优化计划

## 阶段目标

把第一阶段的“可用内容站”升级为低维护、可持续更新、能够反映团队真实需求的内部 AI 工作方法平台。

## P0：稳定运营与发布闭环

1. 自动扫描固定信息源，把候选内容写入灵感收件箱，不直接写入公开数据。
2. 对候选内容执行去重、价值、时效、公司网络环境和可执行性判断。
3. 草稿生成后向飞书发送审核提醒，包含标题、推荐理由、来源和内容中心入口。
4. 审核通过后支持一键同步与妙搭发布。
5. 发布后自动检查轮播、快速匹配、复制、反馈弹窗、图片和移动端布局。
6. 验证失败时停止发布并保留上一个可用版本。

验收标准：连续 4 周按计划运行；未经审核内容为 0；线上交互错误为 0；失败任务不会覆盖可用版本。

## P1：内容质量与团队反馈

1. 将点赞从浏览器本地状态升级为可汇总数据，统计真实有用内容。
2. 汇总“想看什么”反馈，形成待办方向、出现频次和处理状态。
3. 为内容增加“最后核验时间”和“更新说明”的可见表达。
4. 建立每周精选规则：价值、可复制性、团队适用性、时效和近期反馈共同决定排序。
5. 对重复、低价值、仅提供导航的内容自动提示合并或下架。

验收标准：能输出每周内容表现清单；每条新增内容都有明确可带走结果；过期内容不会继续公开展示。

## P2：发现效率与使用体验

1. 优化 Codex 方法合集的搜索、场景筛选和复制路径。
2. 分类页支持稳定分页或分段加载，避免内容增长后页面过长。
3. 增加“最近看过”和“收藏”，优先保存在本地，暂不引入账号系统。
4. 优化移动端卡片密度、中文排版和长标题处理。
5. 为详情页增加相关方法和下一步行动，减少阅读后无从下手。

验收标准：用户从首页到复制可执行内容不超过 3 次操作；主流手机宽度无溢出；分类内容增长到 100 条后仍可快速浏览。

## P3：数据与长期决策

1. 统计页面访问、分类点击、复制、详情查看、点赞和反馈提交。
2. 建立不采集敏感信息的周报，只保留聚合数据。
3. 根据数据决定保留、更新或下架内容，而不是只按发布时间堆积。
4. 评估是否需要账号、团队权限和个性化推荐；没有明确收益前不建设。

验收标准：能够回答“哪些内容被使用、哪些方向值得继续投入、哪些内容应该下架”。

## 推荐实施顺序

1. 定时信息源扫描与飞书审核提醒。
2. 妙搭发布自动化和线上冒烟测试。
3. 点赞与反馈数据闭环。
4. 分类分页、收藏和最近看过。
5. 聚合分析与内容周报。

第二阶段仍坚持小步交付：每一项独立上线、独立验证，不同时改造内容模型、发布链路和首页结构。
