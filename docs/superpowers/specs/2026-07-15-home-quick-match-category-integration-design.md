# 首页快速匹配与分类二级页整合设计

## 目标

保留现有 QIFEI Hero 轮播和原“快速匹配”功能，同时让四个工作目标自然成为分类二级页入口。首页内容数量保持有界，避免分类内容增长后无限拉长页面。

## 页面结构

首页 `/` 按以下顺序展示：

1. QIFEI 精选内容轮播。
2. 快速匹配：按见效时间、主要目标、结果形式和使用门槛推荐 3 条内容。
3. 喂饭式内容发现：搜索、分类筛选、精选/最新排序，以及带“你能带走”和复制操作的内容卡片。
4. 首页内容流最多 18 条；结果标题旁始终提供动态“查看更多”入口。“全部”进入 `/discover`，选中分类后进入对应 `/category/<slug>`。

全量发现页 `/discover` 保留搜索、分类筛选、排序和全量内容网格，不再重复首页快速匹配区块。

分类页 `/category/[slug]` 承载该分类的完整内容，并提供四个分类之间的切换和返回全部发现入口。

## 交互规则

- “主要目标”按钮只更新推荐结果，不立即离开首页。
- 分类入口独立显示，文案包含当前分类名称和真实内容数量。
- 选择其他主要目标后，分类入口的链接、名称和数量同步更新。
- 快速匹配始终最多返回 3 条推荐，继续复用现有推荐算法。
- 首页内容流最多显示 18 条，但结果数量继续显示完整匹配数。
- 首页分类筛选变化后，“查看更多”文案和目标路由同步变化。
- 键盘、焦点、ARIA 状态和移动端布局继续满足现有无障碍约束。

## 代码边界

- 新建独立 `QuickMatch` React 组件，负责偏好状态、三条推荐和分类入口。
- `DiscoveryExplorer` 负责搜索、分类筛选、排序和内容列表，通过 `limit` 与 `showMoreLink` 区分首页和完整发现页。
- 首页直接组合 `HeroCarousel`、`QuickMatch` 和受限版 `DiscoveryExplorer`。
- 分类映射继续以 `lib/categories.ts` 为唯一来源，不复制 slug 映射。
- 不修改推荐算法、公开数据结构、飞书多维表格字段或同步流程。

## 验收标准

1. 首页可见快速匹配，切换主要目标会更新三条推荐。
2. 首页分类入口指向当前主要目标对应的 `/category/<slug>`，并显示真实数量。
3. 首页不再渲染独立四分类入口区块。
4. `/discover` 不重复快速匹配，只显示全量发现能力。
5. 分类页可在四个分类之间切换，单条内容时不产生异常拉伸。
6. 首页第三屏保留搜索、筛选、排序、带走结果、详情和复制操作，最多呈现 18 条。
7. 首页“查看更多”会随分类筛选指向 `/discover` 或对应分类页。
8. 单元测试、Astro 检查、构建、公开数据校验和 Playwright E2E 全部通过。
