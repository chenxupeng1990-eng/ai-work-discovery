# 内容时效门禁与信息源精选设计

## 目标

在每次公开数据集生成前，确保已发布记录仍有明确价值、来源状态可核验、事实符合当前实际，并且尚未超过人工或 AI 建议的复核期限。UI 验收后，从灵感收件箱维护的信息源中筛出可直接交付给同事的具体方法，写成飞书草稿并通知人工审核。

## 核心原则

1. AI 只提供审查建议和草稿，不自动把内容设为“已发布”。
2. 发布门禁失败即阻断：缺少审查、结论不是“通过”或超过复核期限时，保留上一版公开数据集。
3. 来源站点不是内容交付物。每条草稿必须提炼一个具体方法、可执行步骤和可验证产物，不能只推荐首页链接。
4. 对公司无 VPN 环境明确标注“无需 VPN”“部分资源需要 VPN”或“需要 VPN”。
5. 价值判断以 Codex 使用、飞书 CLI/协作、电商与内容生产为主；VibeCodeIdea 默认不收录，除非出现直接匹配的新内容。

## 飞书内容主表审查字段

| 字段 | 类型 | 可选值或格式 | 用途 |
| --- | --- | --- | --- |
| 信息价值 | 单选 | 高价值 / 可保留 / 低价值 | 判断是否值得团队投入时间 |
| 时效状态 | 单选 | 当前有效 / 需复核 / 已过时 / 无法确认 | 判断内容是否仍然成立 |
| 事实状态 | 单选 | 符合当前实际 / 存在偏差 / 无法确认 | 判断关键步骤、产品状态和依赖是否真实 |
| 核验结论 | 单选 | 通过 / 更新后复核 / 下架 / 待审核 | 发布门禁唯一决策字段 |
| 核验说明 | 文本 | 最多 500 字 | 记录依据、变化和风险 |
| 核验时间 | 日期时间 | ISO 时间 | 记录本轮审查时间 |
| 下次复核时间 | 日期时间 | ISO 时间 | 到期后自动阻断下一次发布 |

以上字段全部属于内部字段，不进入 `content.json`。

## 审查 SOP

1. 读取记录标题、摘要、推荐理由、原始链接、网络要求和更新时间。
2. 打开原始来源或其官方替代来源，确认页面仍可访问、项目仍维护、安装命令与产品能力仍成立。
3. 判断目标团队是否能在当前网络和权限条件下获得可验证结果。
4. 识别重复、泛化或只提供导航价值的内容；没有独立行动价值则标记“低价值”。
5. 输出四项结构化判断、核验说明和复核窗口：高波动信号 7 天，工具与 Skill 30 天，稳定方法与案例 90 天。
6. 只有“信息价值=高价值或可保留”“时效状态=当前有效”“事实状态=符合当前实际”“核验结论=通过”且未到下次复核时间的已发布记录，允许进入公开数据集。

## 信息源精选

扫描灵感收件箱中当前维护的来源：飞书 CLI 专区、Codex 专区、Vibe Coding 雷达、AI HOT，以及已有的设计上下文来源。VibeCodeIdea 只做低优先级差异扫描。

每条入库草稿必须包含：具体标题、一句话摘要、独立推荐理由、推荐轨道、上手时间、使用门槛、VPN 标记、可带走结果、2-5 个标签、来源链接和至少一个可直接复制或执行的内容块。新内容写入“草稿”，关联来源收件箱，最终由人工审核。

## 验收标准

- 未审查、已过时、无法确认或过期记录会使同步在替换公开数据集前失败。
- 当前已发布记录完成一轮来源核验并写回审查字段。
- 信息源扫描只创建人审草稿，不自动发布。
- 新草稿满足统一摘要规范并明确网络要求。
- Bot 成功发送审核提醒，用户可从消息打开内容中心。
